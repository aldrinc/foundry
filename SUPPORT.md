# Support

Use the right path so issues land in the right queue.

## Bugs and regressions

Open a GitHub issue with the bug report template and include:

- Foundry version
- operating system
- deployment mode
- reproduction steps
- expected behavior
- relevant logs or screenshots

## Feature requests

Open a GitHub issue with the feature request template and describe:

- the workflow you are trying to support
- the current limitation
- the proposed behavior
- any alternatives you considered

## Security issues

Do not file public issues for security reports. Follow [SECURITY.md](SECURITY.md).

## Licensing and self-hosting questions

Start with [LICENSE](LICENSE), [LICENSING.md](LICENSING.md), and the component READMEs. If the question is still unresolved, open a GitHub issue with enough context to route it correctly.
